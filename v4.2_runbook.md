# PhantomClaw v4.2 Ops Runbook

Purpose: map `/health/ops` reason codes to immediate operator actions so you can answer "safe to trade now?" in under 15 seconds.

---

## 1) Fast Triage (15 seconds)

1. Check `overall_status` and `overall_reason_code` (`/health/ops` or Telegram `/status` Ops block).
2. If `overall_status=GREEN`: continue normal operation.
3. If `overall_status=AMBER`: switch to `OBSERVE` if issue persists beyond 2-5 minutes.
4. If `overall_status=RED`: switch to `OBSERVE` immediately; use `HALT` if auth/contract/execution safety is uncertain.

---

## 2) Mode Decision Rules

1. `GREEN`: keep current mode.
2. `AMBER` with transient infrastructure issues: keep mode, monitor.
3. `AMBER` persistent (>5 min): `/mode observe`.
4. `RED` due to auth/contract/queue stuck: `/mode observe` now.
5. `RED` with unknown cause or repeated execution anomalies: `/halt`.

---

## 3) Reason Code Actions

| Reason Code | Meaning | Immediate Action | Escalation |
|---|---|---|---|
| `OPS_HEALTHY` | All critical checks healthy | No action | None |
| `EA_NO_SIGNALS_YET` | EA not seen by bridge yet | Confirm EA attached to chart, Algo Trading ON, WebRequest includes `http://127.0.0.1:8765` | If no first signal in 2 min: `OBSERVE` |
| `EA_SIGNAL_LAGGING` | Signal stream lagging | Check MT5 terminal connectivity and EA logs | If persists >5 min: `OBSERVE` |
| `EA_STALE_SIGNAL` | Signal stream stale | Treat as broken EA-link path | `OBSERVE` now; `HALT` if execution uncertainty |
| `AUTH_DISABLED` | Bridge token auth not configured | Confirm this is intentional for environment | If not intentional, configure token before `AUTO` |
| `AUTH_INTERMITTENT_FAILURE` | Some unauthorized requests in last 5 min | Verify `BridgeAuthToken` (EA) matches `PHANTOM_BRIDGE_AUTH_TOKEN` | If rising failures: `OBSERVE` |
| `AUTH_UNAUTHORIZED` | Frequent unauthorized requests | Stop trusting signal/decision path auth | `OBSERVE` immediately; fix token; `HALT` if unsure |
| `CONTRACT_INTERMITTENT_MISMATCH` | Some contract header mismatches | Align EA `BridgeContractVersion` with bridge major version | If persistent: `OBSERVE` |
| `CONTRACT_MISMATCH` | Frequent contract mismatches | Version contract broken | `OBSERVE` immediately until fixed |
| `DECISION_LOOP_DEGRADED` | Queue/latency degraded | Check provider latency and queue depth | If queue grows continuously: `OBSERVE` |
| `QUEUE_STUCK` | Decision queue likely stuck | Treat as execution pipeline failure | `OBSERVE` now; `HALT` if repeated |
| `AI_NO_DECISIONS_YET` | Signals arriving but no AI decisions yet | Check model/provider health and API key | If persists beyond warmup window: `OBSERVE` |
| `AI_DECISION_STALE` | No recent decisions while signals continue | Check provider status (`/model status`) and bridge logs | `OBSERVE` until decision flow normal |
| `DATA_AGING` | Core data aging | Verify SSE/poll freshness and bridge updates | If aging worsens: `OBSERVE` |
| `DATA_STALE` | Core data stale | Critical telemetry stale | `OBSERVE` now; `HALT` if uncertain |

---

## 4) Incident Workflow

1. Freeze risk: `/mode observe` (or `/halt` for critical uncertainty).
2. Capture context:
   1. `/status` output
   2. dashboard Diagnostics panel
   3. `/health/ops` payload
3. Fix root cause (auth token, contract version, provider, EA connectivity).
4. Wait for `overall_status=GREEN` and stable for at least 2 polling cycles.
5. Resume with `/mode auto` only after stable recovery.

---

## 5) Quick Commands

1. Check state: `/status`
2. Freeze safely: `/mode observe`
3. Emergency stop: `/halt`
4. Resume after recovery: `/mode auto`
