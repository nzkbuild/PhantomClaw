## Phase A тАФ Security & Data Safety

### 1. Redact secrets from diagnostics (#5)
- **What:** Scan `/api/diagnostics` output for keys containing `key`, `token`, `secret`, `password` тАФ replace values with `"***"`
- **Where:** `cmd/phantomclaw/main.go` (diagnosticsSnapshot function)
- **Risk if skipped:** API keys visible in browser to anyone with network access
- **Status:** Done

### 2. SQLite read-only connection for dashboard (#8)
- **What:** Open a second `?mode=ro&_busy_timeout=5000` connection for all dashboard queries
- **Where:** `internal/memory/db.go` + `internal/dashboard/server.go`
- **Risk if skipped:** Dashboard read blocks bridge write during signal processing тЖТ decision timeout
- **Status:** Done

### 3. Queue model switches (#7)
- **What:** Don't switch model while a signal is being processed. Queue the switch and apply after current signal completes
- **Where:** `cmd/phantomclaw/main.go` (llmSwitch function) + `internal/llm/router.go`
- **Status:** Done тАФ `SetPrimaryQueued`, `BeginSignal`, `EndSignal` on `Router`; signal handler wraps; 6 tests with `-race`

### 4. Dashboard basic auth (#6)
- **What:** Optional `dashboard.auth_user` / `dashboard.auth_pass` in config. If set, require Basic Auth on all `/api/*` and `/` routes
- **Where:** `internal/dashboard/server.go` + `internal/config/config.go`
- **Status:** Done

## Phase B тАФ Dashboard UX

### 5. Equity curve chart (#2)
- **What:** TradingView [Lightweight Charts](https://github.com/nicefresh/lightweight-charts) via CDN. Plot cumulative P&L from trade history
- **Where:** `internal/dashboard/assets/index.html` + new `/api/equity` endpoint in `internal/bridge/server.go`
- **Data source:** `SELECT pnl, closed_at FROM trades ORDER BY closed_at` from SQLite

### 6. Dynamic risk thresholds (#1)
- **What:** `/api/snapshot` returns `max_daily_loss`, `max_positions`, `max_drawdown_pct` from config. Dashboard uses these for color thresholds instead of hardcoded `40`/`80`
- **Where:** `cmd/phantomclaw/main.go` (snapshot lambda) + `index.html`

### 7. Provider status panel (#3)
- **What:** Card showing each provider's name, model, status (ok/cooldown/error), and whether it's current primary
- **Where:** `index.html` тАФ reuse data already in `/api/diagnostics`

### 8. Trade analytics (#12)
- **What:** Win rate, average RR, Sharpe ratio, max drawdown, P&L by pair тАФ computed server-side
- **Where:** New `/api/analytics` endpoint in `internal/memory/db.go` + dashboard card

### 9. Stale data indicator (#9)
- **What:** Show `last_signal_time` in snapshot. Dashboard shows red "EA disconnected (X min ago)" if stale > 60s
- **Where:** `internal/bridge/server.go` (track last signal timestamp) + `index.html`

## Phase C тАФ Operational Polish

### 10. Replace polling with SSE (#11)
- **What:** Server-Sent Events endpoint `/api/events` that pushes snapshot + log updates. Fall back to polling if SSE fails
- **Where:** `internal/dashboard/server.go` + `index.html`
- **Benefit:** Live updates without 28,800 daily requests

### 11. Dashboard model switcher (#4)
- **What:** Dropdown in dashboard showing available providers, POST to `/api/model` to switch. Confirmation dialog before applying
- **Where:** `index.html` + new handler in `internal/dashboard/server.go`

### 12. Provider unknown state (#13)
- **What:** New providers start as `"unknown"` status, transition to `"ok"`/`"error"` after first call
- **Where:** `internal/llm/router.go`

### 13. Log tail optimization (#10)
- **What:** For log files > 10MB, seek to last 10MB before scanning. Prevents slow dashboard on long-running instances
- **Where:** `internal/logging/query.go`

### 14. Dashboard port in config (#14)
- **What:** Add `dashboard.host` and `dashboard.port` to config instead of hardcoded `8080`
- **Where:** `internal/config/config.go` + `cmd/phantomclaw/main.go`
